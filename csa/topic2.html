Taken Notes:

-A class is like a blueprint that’s used to create objects with the same behaviors and attributes.

-Objects are instances of classes.

-Each object’s behaviors/methods and attributes are defined by the class it was made from.

-More than 1 object can be created using a class.

-Each object can be different from each other.

-Constructors are used to initialize the attributes of an object.

-Constructors must have the same name as the class.

-A class also has methods. While attributes are things like “color,” these are actions like “running.”

-Creating a constructor (example):

- - - private String color;

- - - private int number;

- - - public Something(String variable1, int variable2){

- - - - - - color = variable1;

- - - - - - number = variable2;

- - - }

-}

-Creating an object (example): ClassName name = new ClassName(variable1, variable2);

-We can have more than one constructor for an object, which is called overloading the constructor.

-While creating an object, if we don’t set any initial values for the constructors, the values will be set to null.

-We don’t put anything like “int” or “void” in front of a constructor, unlike a method.

-Putting “void” in front of the constructor name would make it so that it doesn’t return anything.

-When there is “this” in front of a variable (written like this.variable), the program selects the uppermost variable with the given name.

-Static methods are called using the dot operator along with the class name. Static methods are those that belong to a class instead of objects of a class, they can’t be used by the objects, only the classes themselves.

-Every primitive data type has a corresponding wrapper class.

-A string can be created using the primitive data type or the wrapper class.

-Using Primitive Data:

-String ha = “Hello”;

-Using Class:

-String haha = new String(“Hello”);

-or

-String hahaha = new String(ha);

-Using String class to make a string makes the created string an object.

-String objects are immutable, methods that act upon String objects don’t change the String itself.

-+ joins the two written String values together. += also does the same. They also redefine other values (like integer or double etc.) to String. += adds all of them to the previous String value. + does this only after the other data type touches a String. If the code is written like (“hah” + 1 + 2), the outcome would be (hah12), but if it was written like (1+2+“hah”), the outcome would be (3hah). The placement matters if we are using +.

-Example:

-String str = “haha”;

-str = str + “hahaha; //turns str to “hahahahaha”

-String str1 = “haha”;

-str1 += 123 + “kak”; //turns str1 to “haha123kak”

-\ can be used to add stuff like \ and *, or it can be used to add a new line by writing \n.

-Example:

-System.out.println(“\”” + “\*”); //prints “*

-stringObject.indexOf(String) returns the location of the requested element in the String.

-Example:

-String hah = new String(“hahnbbn”);

-System.out.println(hah.indexOf(“a”)); //prints 1 because “a” is at location 1 in the String “hahnbbn”

-stringObject.length() returns the number of characters in a string as an integer value.

-stringObject.substring(from int, to int) returns the string of characters between numbers written inside, exclusive.

-stringObject.substring(from int) returns the string of characters from int onwards.

-Example:

-String str = “naber”;

-System.out.println(str.length()); //returns 5 as integer

-System.out.println(str.substring(1,3)); //returns “ab” as String

-System.out.println(str.substring(1)); //returns “aber” as String

-stringObject.equals(String) compares the two variables and returns a boolean value to answer whether they are equal or not.

-stringObject.compareTo(String) compares the two variables and returns an integer value depending on the comparison.

-Example:

-String str = pal;

-System.out.println(str.compareTo(“palm”)); //returns -1

-Integers can also be created as objects.

-Example:

-Integer a = new Integer(123);

-intObject.intValue() returns the value of the object, which is Integer class, as the primitive data type int.

-Doubles can also be created as objects.

-Example:

-Double a = new Double(123.0);

-doubleObject.doubleValue() returns the value of the object, which is Double class, as the primitive data type double.

-Double and Integer are wrapper classes (they can be used to create an object that corresponds to the class), while double and int are primitive data types.

-Autoboxing is the program setting the given value automatically to the wrapper class object.

-Example:

-Integer haha = 128; //Even though the code doesn’t have “new Integer(128),” autoboxing makes it so that this gives the same outcome as it having “new Integer(128).”

-Autoboxing turns the int value to Integer.

-Unboxing is the program setting the value of a wrapper class object to the given variable.

-Example:

-Integer haha = new Integer(128);

-int hahahaha = haha;

-int hahhaa = haha + hahahaha + 12089;

-Unboxing turns the Integer value to int; it’s the opposite of autoboxing.

-Math class helps us do math in the code.

-Math class is a static class. This means every method in the Math class can be used like Math.methodName().

-Methods Math class has:

-Math.abs(intValue); //gives the absolute value of an int.

-Math.abs(doubleValue); //gives the absolute value of a double.

-Math.pow(double base, double exponent); //gives the power exponent of the double base.

-Math.sqrt(doubleValue); //gives the square root of a double value.

-Math.random(); //gives a random double value between 0 and 1, exclusive (0.0 <= value < 1.0).

-Example usage of Math.random():

-if (Math.random() < 0.5){}

-This makes it so that the if statement has a 50% chance to work.

-(int) (Math.random() * 11) + 20;

-In this code, (int) (Math.random() * 11) gives a random int value between 0 and 11, exclusive (0 <= value < 11), while + 20 makes it so that the value is between 20 and 31 (20 <= value < 31).

-* decides how much the possible values can differ from each other, while + selects the value's starting point.

-Integer.MIN_VALUE and Integer.MAX_VALUE don't have () at the end.

-Some methods need arguments while some don’t.

-An argument is what’s inside the ().

-A constructor with no parameters is also called a no-argument constructor.

-An object variable holds a reference to an object. This means that it helps us find the place the object is located in the memory.

-Actual parameters are the values we write while creating an object. They are the arguments.

-Formal parameters are the variables inside the constructor.

-Example:

-public Hello (int day, int hour, String what){

-}

-Hello krm = new Hello(12,18,”haha”);

-The 12, 18, and the “haha” are actual parameters.

-int day, int hour, and String what are the formal parameters.

-When creating an object, the values of the written actual parameters are copied to the formal parameters.

-Methods inside the same class can call each other by just writing nameOfMethod().

-The attributes in a class are also named instance variables.

-If we print an object by itself, it will print out its toString method. The toString method prints some text that’s not that understandable. We can change this toString method by using override (this will be discussed in later topics). Everything returned from a toString method turns to a string automatically.

-Example:

-Writing System.out.println(objectName); would print out its toString method.
